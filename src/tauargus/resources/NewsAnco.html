News

12 January 2012 As a test non-additive tables can be imported. It will be tricky to apply the suppression based on the optimisation routines, but the Hypercube is more flexible in this.
28 June 2011 Experiment on request of Luigi: Virgili
1. The audit-routine can also be used before cell suppression.
2. After the cell suppression the status of a secondary can be changed manually.
26 April 2011 Linked tables with modular is now possible
When a set of tables can be seen as as subset of a larger cover table, this set can now be protected.
It is a restriction that the cover table has not more than 4 dimensions.
The problem can also be solved using the hypercube.
10 February 2011 A first version of the incomplete tables option is available.
If not all blocks of a table are filled (e.g different detail in parts of a hierarchical table) τ-ARGUS can now handle this.
All 'live' empty cells will be automatically converted into protected cells with value 0.
Then this table will be protected.
Temporarily this only works with tabular input including the status. This version is needed for protecting linked tabels
1 February 2011 The three singletons-rules (familiar from the modular procedure) are now also available for the full optimisation.
This allows for the additional singleton protection in optimal.
The protection is achieved by adding additional cells (combination of singletons) to the problem

Anco features

Linked tables via the modular protection is available. Both starting from fixed format microdata and from tabular input.
When starting with tables use the option "Open Table set" to load the tables.
When starting with microdata just specify a set op tables. You can then still use global recodings to modify the tables.
Via Modify|Linked tables modular you can start the linked tables procedure. There is a standard automatic procedure to generate the cover table, but it can be done manually ss well.
In batch the command is LINKMOD without additional parameters
A JJ-file can be protected directly. Via the menu "Modify|Protect JJ Format" you will be prompted for the name of the inputfile, the output file and a time limit
In batch the command is JJ followed by the input, output file and optional the time limit (10 min. by default)
Via "Output|Save Table" a table can be stored as a JJ-file.
The solver options can be modified, but use with care.
The distance funtion can be used as a cost function. see metadata file and the specify tables screen
The marginal suppression is available, finding secondaries always in the margins. The Singleton preprocessing is still available, but not recommended.
The Q-parameter is available for the PQ-rule.
In the optimisation routines the cost-function is scaled at the interval 0,20000. This max can be changed when specifying the tables (MSC).
